# Security Policy

## Supported versions

Security fixes are applied to the latest revision of `main`. Older revisions are not supported.

## Reporting a vulnerability

Use GitHub private vulnerability reporting under the repository Security tab. Do not disclose a suspected vulnerability in a public issue.

Include the affected component, impact, reproduction steps, and any known mitigation. Remove credentials, meeting content, and personal data from the report.

If private vulnerability reporting is unavailable, contact the maintainer privately through the contact methods on their GitHub profile before sharing technical details.
